diag_style command

Syntax: 

diag_style style keyword value keyword value ...
  • style = cluster or cluster2d or cluster3d or eprof3d or energy or energy2d or energy3d
  • zero or more keyword/value pairs may be appended 
    keyword = delt or logfreq or stats
  delt values = delta
    delta = time increment between evaluations of the diagnostic (seconds)
  logfreq values = N factor
    N = number of repetitions per interval
    factor = scale factor between interval
  stats values = yes or no
    yes/no = provide output to stats line
  • see doc pages for individual diagnostic commands for additional keywords. These diagnostic-specific keywords must come after all of the standard keywords

Examples: 

diag_style cluster stats no delt 1.0 
diag_style eprof3d stats no delt 0.01 logfreq 7 10.0
diag_style energy2d

Description:

This command invokes a diagnostic calculation. Currently, diagnostics can only be defined for on-lattice applications. See the app_style command for an overview of such applications. As described by the app_style command, on-lattice applications use one of 3 styles of lattice: general, 2d, or 3d. For several of the diagnostic styles, you must use a style that matches the application's lattice.

The diagnostics currently available are:

  • cluster = grain size statistics for general lattices
  • cluster2d = grain size statistics for 2d lattices
  • cluster3d = grain size statistics for 3d lattices
  • eprof3d = site energy as function of distance from sector boundary for 3d lattices
  • energy = compute energy of entire system for general lattices
  • energy2d = compute energy of entire system for 2d lattices
  • energy3d = compute energy of entire system for 3d lattiecs

Diagnostics may provide one or more values that are appended to other statistical output and printed to the screen and log file via the stats command. We call this stats output. In addition, the diagnostic may optionally write more extensive output to files if requested by diagnostic-specific keywords.

The stats keyword controls whether or not the diagnostic appends values to the statistical output. If stats is set to yes, then the delt and logfreq keywords may not be used, as the output interval is controlled by the stats command. If stats is set to no, then the delt keyword specificies Delta, the interval of time between each diagnostic calculation. Using the logfreq keyword will cause the diagnostic to run at varying intervals during the course of a simulation. There will be N outputs per interval where the size of each interval scales up by factor each time. Delta is the time between outputs in the first (smallest) interval.

For example, this command 

diag_style eprof3d stats no delt 0.1 logfreq 7 10.0

will perform its computation at these times: 

t = 0, 0.1, 0.2, ..., 0.7, 1, 2, ...., 7, 10, 20, ....

This command 

diag_style enegy eprof3d stats no delt 0.1 logfreq 1 2.0

will perform its computation at these times: 

t = 0, 0.1, 0.2, 0.4, 0.8, 1.6, ...

Restrictions: none

Related commands:

stats

Default:

The stats setting is yes.
