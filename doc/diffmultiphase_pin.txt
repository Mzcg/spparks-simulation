diffusion/multiphase pin command :h3

[Syntax:]

diffusion/multiphase pin id :pre

id = integer value; declares phase label and pins the phase

This command must follow the 'create_sites' command in the input script.

[Examples:]

###########################################################################
# Example: declaring pinned phase #########################################
###########################################################################
# This example defines three phases: 1,2,3
#    phase 1 is pinned
#    weight between phases 2 and 3 set to 0.5; 
#    weight between phases 1 & 2 and phases 1 & 3 defaults to 1.0
#
diffusion/multiphase pin 1
diffusion/multiphase phase 2
diffusion/multiphase phase 3
#
# Pairwise phase weights are optional; default 1.0
# Need only input weight once for pair (i,j); symmetry 
#    is enforced in app; same weight is internally 
#    assigned to pair(j,i)
# diffusion/multiphase weight &ltdouble value&gt pair &ltint phase_1,int phase_2&gt 
diffusion/multiphase weight 0.5 pair 2 3
########################################################################### :pre

[Description:]


This is an optional command used by "app_style
diffusion/multiphase"_app_diffusion_multiphase.html to specify a particular phase as
pinned; the app simulates diffusion by swapping sites; however, sites that are
pinned are not allowed to swap thereby pinning them in place.  The energy at a
pinned site does not contribute to the total system energy since the site can
never swap with neighbors.  However, pinned sites do contribute to the energy of
neighboring sites in the usual way by summing the bond energies of all
neighboring sites which may include some pinned neighbors.  

[Restrictions:]

This command may only be used by "app_style
diffusion/multiphase"_app_diffusion_multiphase.html and is not applicable to any other apps.

[Related commands:]

"app_style diffusion/multiphase"_app_diffusion_multiphase.html 

"diffmultiphase_weight"_diffmultiphase_weight.html

"diffmultiphase_phase"_diffmultiphase_phase.html

[Default:] 

No pinned phases.
