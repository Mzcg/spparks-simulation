phasesep phase command

Syntax: 

phasesep phase id

id = integer value; declares phase label

This command must follow the 'create_sites' command in the input script.

Examples: 

###########################################################################
# Example: declare phase label  ###########################################
###########################################################################
# This example defines three phases: 1,2,3
#    phase 1 is pinned
#    weight between phases 2 and 3 set to 0.5; 
#    weight between phases 1 & 2 and phases 1 & 3 defaults to 1.0
#
phasesep pin 1
phasesep phase 2
phasesep phase 3
#
# Pairwise phase weights are optional; default 1.0
# Need only input weight once for pair (i,j); symmetry 
#    is enforced in app; same weight is internally 
#    assigned to pair(j,i)
# phasesep weight <double value> pair <int phase_1,int phase_2> 
phasesep weight 0.5 pair 2 3
###########################################################################

Description:

This is a required command used by app_style phase/separation to specify integer phase labels for all phases in the simulation.

Restrictions:

This command may only be used by app_style phase/separation and is not applicable to any other apps.

Related commands:

app_style phase/separation

phasesep_weight

phasesep_pin

Default:

None
